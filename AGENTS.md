# Agent Notes

이 문서는 CheapestOilFinder 저장소에서 Codex가 따라야 할 규칙과 현재 합의된 구현 방향을 기록합니다.  
작업 전에 반드시 읽고, 기능을 구현하거나 구조를 바꾸면 관련 문서를 함께 갱신합니다.

## 문서 언어 규칙

- `README.md`와 `AGENTS.md`는 한국어로 작성합니다.
- 새 문서와 기존 문서 수정도 기본적으로 한국어를 사용합니다.

## 현재 언어 규칙

- 앱 구현 코드는 Kotlin을 기본으로 합니다.
- 기존 Java 소스는 제거했고, 신규·수정 코드는 Kotlin으로 작성합니다.

## 담당 범위

- 현재 Codex의 주 담당 범위는 Android 프론트엔드입니다.
- 백엔드 서버, 오피넷 수집기, PostgreSQL 스키마 구현은 현재 범위 밖입니다.
- 프론트엔드는 오피넷 등 외부 유가 API를 직접 호출하지 않습니다.
- 주유소, 경로, 추천 결과 데이터는 백엔드 서버에서 받아오는 구조를 전제로 합니다.
- 백엔드가 준비되기 전에는 더미 데이터 또는 placeholder repository로 UI 흐름을 검증합니다.

## 현재 지도·리스트 규칙

- 현재 위치 화면은 진입 시 기기 GPS를 읽고 주변 주유소를 자동으로 불러옵니다.
- 현재 위치 화면의 `현 위치로 설정` 버튼은 현재 위치 좌표를 기준으로 반경 5km 주유소를 백엔드에 요청합니다.
- 현재 위치 화면의 주유소 조회는 `radiusMeters=5000` 형식을 사용합니다.
- 현재 위치 화면의 `GPS` 버튼은 현재 위치를 강제로 재수신하는 버튼이며, 화면 하단 리스트와 분리된 고정 위치에 항상 보입니다.
- `GPS` 버튼은 누르는 즉시 캐시된 현재 위치로 먼저 화면 중심을 맞추고, 재수신이 성공하면 새 위치로 다시 중심을 맞춥니다.
- 이전 좌표와 새 좌표의 차이가 10m 미만이면 백엔드 재요청을 생략합니다.
- `GPS` 버튼은 지도보다 위, 주유소 리스트와 정보 패널보다 아래 레이어에 둡니다.
- 화면 오른쪽 중앙의 `+`/`-` 버튼은 4단계 프리셋 줌만 이동시킵니다.
- 손가락 핀치는 기존처럼 자유롭게 확대·축소할 수 있습니다.
- 현재 위치 화면의 설정값은 `UserPreferenceManager`가 SharedPreferences에서 읽고, `CostCalculator`가 이동비용·예상 주유비·예상 총비용을 계산합니다.
- 현재 위치 화면의 주유소 리스트는 화면 하단의 `최소화 / 절반 / 전체` 3단계 패널입니다.
- `최소화`는 숨김이 아니라 화면 하단에 접혀 있는 상태입니다.
- `절반`은 화면 하단 절반 높이의 상태입니다.
- `전체`는 화면을 가득 채운 상태입니다.
- 첫 주변 조회 성공 시 리스트는 반드시 `최소화` 상태로 시작합니다.
- 리스트 상단의 손잡이 영역에서만 확장·축소 제스처를 받습니다.
- 리스트 본문은 스크롤만 담당하고, 지도 터치와 간섭하지 않아야 합니다.
- 리스트 또는 상세 패널이 표시되면 `현 위치로 설정` 버튼은 숨깁니다.
- GPS 버튼은 리스트나 상세 패널이 떠 있어도 항상 보여야 합니다.
- 주유소 마커와 유가 말풍선은 화면 아래쪽에 있는 항목이 더 높은 우선순위를 가져가도록 화면 기준으로 정렬합니다.
- 현재 위치 화면은 사용자가 선택한 유종 가격이 0원이거나 없으면 해당 주유소를 지도와 리스트에서 제외합니다.
- 주유소 리스트에는 저장되지 않는 임시 유종 드롭다운이 있으며, 이 선택값은 현재 위치 화면의 비용 계산, 마커 필터, 지도/리스트의 유종 라벨에 모두 반영합니다.
- 주유소 리스트의 임시 유종 드롭다운은 헤더 우측 상단에 배치합니다.
- 주유소 가격 박스는 줌 레벨에 따라 5개 / 20개 / 50개 / 전체 구간으로 나뉩니다.

## 현재 화면 상태 계약

| 상태 | 의미 |
| --- | --- |
| `MINIMIZED` | 화면 하단으로 접혀 있는 상태 |
| `COLLAPSED` | 화면 하단 절반 높이로 올라온 상태 |
| `EXPANDED` | 화면 전체를 덮는 상태 |
| `HIDDEN` | 내부 비표시 상태 |

- `HIDDEN`은 사용자가 직접 조작하는 3단계 상태가 아닙니다.
- 리스트를 수정할 때는 최소화·절반·전체 전환, 본문 스크롤, 지도 터치, 뒤로가기 동작을 함께 확인합니다.

## 현재 위치 화면 흐름

1. 화면 진입 시 위치 권한과 GPS 위치를 확인합니다.
2. 위치 수신에 성공하면 현재 위치 마커와 5km 원을 표시합니다.
3. `GET /api/stations/nearby`로 주변 주유소를 자동 조회합니다.
4. 응답을 지도 마커와 리스트에 반영합니다.
5. 첫 성공 응답 후 리스트는 반드시 하단 `최소화` 상태로 시작합니다.
6. 설정 화면에서 저장한 유종·연비·1회 주유량은 SharedPreferences에서 읽어 주변 조회 요청과 비용 계산에 반영합니다.
7. 현재 위치 화면의 주유소 리스트는 선택 유종 가격, 이동비용, 예상 총비용을 중심으로 표시합니다.

## 설정 화면 규칙

- 설정 화면은 유종, 차량 연비, 1회 주유량을 입력받습니다.
- 유종 선택지는 고급휘발유, 보통휘발유, 경유 3개만 노출하며 LPG는 선택할 수 없습니다.
- 기본 저장값은 `GAS_LOW`, `12.0km/L`, `30.0L`입니다.
- 입력값 검증 실패 시 `Toast` 또는 입력창 에러로 안내합니다.
- 저장은 `UserPreferenceManager`를 통해 SharedPreferences에 기록합니다.

## 비용 계산 규칙

- 선택 유종은 현재 위치 화면의 저장된 설정값을 사용합니다.
- 유종별 가격 매핑은 아래와 같습니다.
  - `GAS_HIGH` → `premiumGasolineWon`
  - `GAS_LOW` → `regularGasolineWon`
  - `DIESEL` → `dieselWon`
  - `LPG` → `lpgWon`
- 해당 유종 가격이 `null`이거나 0 이하이면 계산 대상에서 제외하거나 “해당 유종 가격 없음”으로 표시합니다.
- 이동비용 계산식은 `(주유소까지의 이동거리_km / 차량연비_km_per_l) * 선택유종_리터당가격 * 2`입니다.
- 예상 주유비 계산식은 `선택유종_리터당가격 * 1회주유량_L`입니다.
- 예상 총비용 계산식은 `이동비용 + 예상 주유비 - discountAmount`이며, `discountAmount`는 기본값 0입니다.
- 거리 값이 meter 단위로 오면 km로 변환해서 사용합니다.
- 거리 값이 없으면 더미 값을 넣지 말고 TODO 주석으로 처리합니다.

## 주유소 정보 패널 규칙

- 주유소 마커나 리스트 항목을 누르면 같은 상세 패널을 엽니다.
- 상세 패널에는 브랜드 로고, 상호명, 도로명주소, 전화번호, 유가 정보, 주유소까지 거리, 이동비용, 예상 주유비, 예상 총비용을 표시합니다.
- 전화번호가 비어 있으면 임시 문구를 표시합니다.
- 0원 유가는 표시하지 않습니다.
- 상세 패널은 하단에서 올라오며, 배경 지도는 어둡게 만들지 않습니다.

## 경로 기능 상태

- `POST /api/stations/route` 요청 인터페이스와 repository 메서드는 존재합니다.
- 백엔드가 `route.routePolyline`을 내려주면 그대로 Kakao `RouteLine`으로 그립니다.
- 실제 응답에 `routePolyline`이 없을 때는 프론트가 출발지-도착지 직선 fallback을 그려 경로를 최소한 표시합니다.
- `RouteInfo`와 `MapController.showRoute()` 인터페이스는 존재합니다.
- 프론트는 현재 위치와 선택 주유소 좌표를 백엔드에 보내고, 백엔드는 네이버 Directions 5를 호출합니다.
- 프론트는 백엔드가 반환한 WGS84 좌표열을 Kakao `RouteLine`으로 그리는 방향을 유지합니다.

## 백엔드 API 기준

- 주변 조회: `GET /api/stations/nearby`
- 상세 조회: `GET /api/stations/{stationId}`
- 경로 조회: `POST /api/stations/route`
- API 계약과 호출 예시는 `docs/FRONTEND_BACKEND_API_GUIDE.md`에서 관리합니다.
- 예전 `/api/stations/search/nearby` 경로는 사용하지 않습니다.
- Retrofit의 query list는 필요 시 Kotlin wildcard를 억제합니다.
- 현재 상세 API의 `phone` 값이 비어 있는 문제는 백엔드 이슈로 관리합니다.

## 비밀 정보 규칙

- API 키, 네이티브 앱 키, 서명키, 토큰은 소스 코드에 직접 작성하지 않습니다.
- 로컬 민감 정보는 `secrets/` 폴더에 둡니다.
- 카카오맵 네이티브 앱 키는 `secrets/kakao_native_app_key.txt`에서 읽습니다.
- `secrets/*.txt`, `secrets/*.properties`, `secrets/*.key`, `secrets/*.jks`는 Git에 올리지 않습니다.
- `keystore.properties`, `signing/*.jks`, `signing/*.keystore`도 Git에 올리지 않습니다.
- release 빌드에는 민감한 API 키를 포함하지 않습니다.
- 키 해시나 서명 관련 디버그 로그는 개발 중에만 허용하고, 운영 빌드에서는 제한해야 합니다.
- `secrets/backend_base_url.txt`를 바꾸면 다음 debug 또는 release Gradle 빌드가 그 값을 다시 읽어 `BuildConfig.BACKEND_BASE_URL`에 반영합니다.

## 프론트엔드 아키텍처 방향

- 지도 SDK 세부 구현은 `KakaoMapController` 안에 모읍니다.
- Activity는 화면 이벤트와 상태 전달에 집중합니다.
- 백엔드 데이터 접근은 repository 계층으로 분리합니다.
- 백엔드 연동은 비동기 네트워크 호출을 전제로 설계합니다.
- 지도, 주유소, 경로, 추천 계산 모델은 SDK 전용 타입에 직접 묶지 않습니다.
- 주유소 마커, 경로 polyline, 추천 결과 오버레이는 지도 SDK 위에 얹는 렌더링 계층으로 확장합니다.

## 작업 방식

- 작업을 시작하기 전에 `README.md`와 `AGENTS.md`를 읽고 현재 구현 방향과 제약을 확인합니다.

## 커밋 메시지 규칙

- 커밋 전에는 이번 변경이 문서, 기능, 버그 수정 중 어디에 속하는지 먼저 분류합니다.
- 커밋 메시지는 기본적으로 다음 형식을 따릅니다.

```text
docs: 문서 위주 변경 요약

- 세세한 변화내용1
- 세세한 변화내용2
```

```text
feat: 기능 구현 위주 변경 요약

- 세세한 변화내용1
- 세세한 변화내용2
```

```text
fix: 버그 수정 위주 변경 요약

- 세세한 변화내용1
- 세세한 변화내용2
```

- 문서 위주의 변경은 `docs:`, 기능 구현 위주의 변경은 `feat:`, 버그 수정 위주의 변경은 `fix:`를 사용합니다.
- 커밋 본문에는 핵심 변화 1줄과 세부 변경 bullet 2개 이상을 적습니다.

## 현재 위치 화면 추가 규칙

- 현재 위치 화면의 주유소 마커는 줌 레벨에 따라 유가 박스 표시 수준이 달라집니다.
- 확대가 충분하면 지도 말풍선은 사용자가 선택한 유종 가격을 우선 표시합니다.
- 상세 유가는 상세 패널에서 모두 확인합니다.
- 더 줌아웃하면 유가 텍스트 박스를 모두 숨기고 주유소 아이콘만 표시합니다.
- 주유소 가격 박스는 줌 11~12에서 단계적으로 밝아지고, 13~15에서 완전히 보이며, 16 이상에서는 더 큰 버전으로 보여야 합니다.
- 주유소 가격 박스는 줌아웃 단계에 따라 서서히 투명해졌다가, 줌 10 이하에서는 사라져야 합니다.
- 주유소 가격 박스는 흰색 배경과 검은 글자로 표시합니다.

## UI 보조 규칙

- 주유소 리스트나 정보 패널이 보이는 동안에는 `현재 위치로 설정` 버튼을 숨기고, 패널이 완전히 숨겨졌을 때만 다시 나타나야 합니다.
- 주유소 리스트의 드래그 제스처는 상단의 작은 가로 손잡이에서만 받아야 합니다.
- 실제 드래그 터치 영역은 손잡이보다 더 두껍고 가로로 길게 잡아, 가운데뿐 아니라 가장자리 쪽에서도 드래그가 잡히게 합니다.
- 주유소 리스트 본문은 스크롤만 담당하고, 시트 확장/축소 제스처는 손잡이 터치 영역만 담당해야 합니다.

## 최근 렌더링 메모

- 주유소 말풍선은 겹쳐도 뒤쪽 항목을 강제로 숨기지 않고 그대로 유지합니다.
- 화면 이동 시에는 기존 라벨을 재사용하고 필요한 항목만 추가/삭제해서 깜박임을 줄입니다.
- 화면 기준 정렬은 유지하되, 라벨 경쟁은 끄고 `CompetitionType.None`으로 둡니다.

## GitHub 준비

- `secrets/` 항목, 서명 파일, 빌드 출력물은 Git 히스토리에 포함하지 않습니다.
- `secrets/*.example`과 `secrets/README.md` 같은 안내 파일만 커밋해도 됩니다.


## 경로 표시 규칙

- 주유소를 선택하면 프론트는 현재 위치와 선택한 주유소 좌표를 백엔드 경로 API에 다시 보냅니다.
- 백엔드 응답의 `route.routePolyline`을 Kakao Map 위 경로 선으로 표시합니다.
- 경로 선은 앱 테마 주황색으로 표시하며, 기본 선보다 1.2배 두껍게 렌더링합니다.
- 경로 선을 그린 뒤에는 경로가 화면에 잘 보이도록 카메라를 자동으로 맞춥니다.
- 경로 선은 주유소 마커와 정보 패널과 분리해서 유지하고, 새 주유소를 고르면 갱신합니다.
- 목적지 기준 찾기 화면은 진입 시 GPS를 자동으로 다시 읽어 현재 위치 마커로 먼저 맞춥니다.
- 목적지 기준 찾기 화면의 상단 검색바를 누르면 전체화면 검색 오버레이가 열리고, 검색창에 커서와 키보드가 자동으로 뜹니다.
- 검색 오버레이의 입력은 1초 동안 멈췄을 때 자동완성 후보 목록으로 갱신되며, 후보를 누르면 검색창 문구가 채워집니다.
- 자동완성은 백엔드 `GET /api/places/autocomplete?query=...`를 우선 호출하고, 응답의 `items[]`를 파싱합니다. 실패하거나 비면 로컬 플레이스홀더 후보로 보강합니다.
- 자동완성 후보는 화면 아래 리스트로 3~4개 정도만 보여줍니다.
- 검색창 우측의 🔎 버튼과 키보드 엔터는 백엔드 `POST /api/places/search`를 호출하고, 결과는 검색 오버레이가 닫힌 뒤 별도의 하단 검색 결과 시트에 표시합니다.
- 검색 API 결과는 리스트와 함께 지도 마커 캐시로 저장하고, 지도 카메라가 움직이면 같은 결과를 다시 렌더링합니다.
- 검색 API 응답은 백엔드 `items[]`의 `placeName`, `roadAddressName`, `addressName`, `latitude`, `longitude`를 우선 파싱하되, Kakao 원본 호환을 위해 `x/y`와 `documents/results/places` 배열도 방어적으로 처리합니다.
- 검색 버튼이나 키보드 검색키로 검색을 확정하면 검색 오버레이를 닫고, 결과 마커가 화면에 모두 보이도록 카메라를 자동으로 맞춥니다.
- 전체화면 검색 오버레이는 검색 결과 시트보다 항상 높은 우선순위를 가져야 하며, 검색 결과 시트가 열려 있어도 검색바를 누르면 오버레이가 화면과 입력 포커스를 덮어야 합니다.
- 목적지 검색 결과 카메라 이동은 현재 위치를 포함하지 않고 검색 결과 마커 좌표만 기준으로 맞춥니다.
- 검색 결과는 목적지 검색 오버레이보다 낮은 우선순위의 별도 하단 시트로 보여주고, 지도에는 기본 빨간 점 마커로 표시합니다. 선택된 장소만 빨간 `📍` 목적지 핀 마커로 표시합니다.
- 검색 결과 페이지의 사용자 상태는 주유소 리스트와 같은 `최소화 / 절반 / 전체` 3단계입니다.
- 검색 결과 페이지의 내부 상태명은 `MINIMIZED / COLLAPSED / EXPANDED`로 관리하며, `HIDDEN`은 검색 결과가 없거나 표시하지 않는 내부 비표시 상태입니다.
- 검색 결과 페이지의 리스트 항목을 선택하면, 선택 장소를 전체 화면 중앙이 아니라 상단 기준 약 25% 위치에 맞추고 줌 15로 확대합니다.
- 검색 결과 시트가 `EXPANDED` 상태일 때 리스트 항목을 선택하면, 선택 직후 시트를 `COLLAPSED`로 접어 지도와 선택 마커를 보여줘야 합니다.
- 지도 위 빨간 점 마커를 선택할 때는 카메라를 움직이지 않고 선택 마커만 빨간 `📍` 핀으로 바꿉니다.
- 목적지 기준 찾기 화면의 GPS 버튼과 `+`/`-` 줌 버튼은 현재 위치 화면과 같은 위치와 모양을 사용합니다.
- 목적지 기준 찾기 화면의 `+`/`-` 버튼은 4단계 프리셋이 아니라 지도 중심을 유지한 채 현재 Kakao 줌 레벨에서 1단계씩 이동합니다.
- 목적지 기준 찾기 화면의 GPS와 줌 버튼은 검색 결과 시트가 없을 때 하단에 가깝게 두고, 검색 결과 시트가 `MINIMIZED` 또는 `COLLAPSED`일 때 시트 손잡이 위로 함께 이동합니다.
- 검색 결과 시트가 `EXPANDED` 상태일 때 GPS와 줌 버튼은 시트를 따라가지 않고 검색 결과 시트 뒤에 가려져야 합니다.
- 목적지 기준 찾기 화면의 `목적지로 설정` 버튼은 장소 선택 전에는 `GONE` 상태로 숨깁니다.
- 장소가 선택되면 `목적지로 설정` 버튼을 표시하고, 검색 결과 시트가 `MINIMIZED` 또는 `COLLAPSED`일 때 시트 손잡이 위에 겹치지 않게 배치합니다.
- 검색 결과 시트가 `EXPANDED` 상태일 때는 `목적지로 설정` 버튼이 시트에 가려져 보이지 않아야 합니다.
- `목적지로 설정` 버튼은 현재 GPS 위치와 선택 목적지를 이용해 `POST /api/stations/route`를 호출하고, 응답의 `route.routePolyline`을 지도 경로선으로 표시합니다.
- 목적지 경로 카메라는 검색 결과 시트용 하단 패딩을 제거한 뒤 전체 지도 화면 중앙 기준으로 맞춰야 합니다.
- 목적지를 확정하면 선택 목적지만 🚩 마커로 남기고 나머지 검색 결과 빨간 점은 제거합니다.
- 목적지 확정 후에는 `목적지로 설정` 버튼과 검색 결과 시트를 숨깁니다.
- 목적지 확정 상태에서 뒤로가기 입력이 들어오면 경로선과 🚩 확정 마커를 해제하고, 확정 직전의 검색 결과 시트 상태와 선택 목적지 핀 상태로 복구합니다.
- 목적지 경로선 중간에는 총 이동거리 라벨을 `m` 또는 `km` 단위로 표시하며, 라벨은 우측 공간이 부족하면 좌측으로 펼쳐지게 배치합니다.
- 향후 목적지 기반 주유소 추천은 확정된 경로선 주변 5km 이내 주유소 조회, 경로 수직 사영 거리와 유가를 이용한 1차 top 10 필터, 경유 증가거리와 연비 기반 비용 계산을 통한 2차 정렬 순서로 확장합니다.
- 목적지 기준 찾기 화면의 검색바는 뒤로가기 버튼 옆에 붙여 배치합니다.
