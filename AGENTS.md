# Agent Notes

이 문서는 CheapestOilFinder에서 작업하는 에이전트가 따라야 할 최신 규칙과 구현 기준을 정의합니다. 과거 변경 이력을 누적하는 문서가 아니며, 동작이 바뀌면 기존 기준을 교체해 항상 하나의 일관된 현재 상태만 유지합니다.

## 작업 시작 규칙

- 모든 작업 전에 `README.md`와 `AGENTS.md`를 읽습니다.
- 관련 코드와 문서를 먼저 확인하고 현재 동작을 추측하지 않습니다.
- 기존 사용자의 변경이나 관련 없는 작업 트리 변경을 되돌리지 않습니다.
- 검색은 `rg`, 파일 목록은 `rg --files`를 우선 사용합니다.
- 수동 편집은 `apply_patch`를 사용합니다.
- 신규·수정 앱 코드는 Kotlin으로 작성합니다.

## 담당 범위

- 주 담당 범위는 Android 프론트엔드입니다.
- 백엔드 서버, 오피넷 수집기, PostgreSQL 스키마 구현은 현재 범위 밖입니다.
- 프론트엔드는 오피넷이나 네이버 Directions 5를 직접 호출하지 않습니다.
- 주유소·상세·경로·추천 데이터는 백엔드 API를 통해 받습니다.
- 백엔드 데이터 접근은 repository 계층으로 분리하고 비동기 호출을 사용합니다.
- 지도 SDK 세부 구현은 `KakaoMapController`에 모읍니다.
- Activity는 화면 이벤트, 상태 전환, repository와 지도 컨트롤러 사이의 조정에 집중합니다.
- 앱 모델은 Kakao SDK 전용 타입에 직접 종속시키지 않습니다.

## 문서 관리 규칙

- `README.md`와 `AGENTS.md`는 항상 한국어와 UTF-8로 작성합니다.
- 사용자 기능, 현재 구현 상태, 빌드·설정 방법이 바뀌면 `README.md`를 갱신합니다.
- 작업 규칙, 보안 규칙, 아키텍처 전제, UI 상태 계약이 바뀌면 `AGENTS.md`를 갱신합니다.
- 새 동작을 문서 끝에 “최근 규칙”으로 반복 추가하지 않습니다.
- 같은 주제의 기존 섹션을 찾아 교체하거나 통합합니다.
- 구현 완료 기능과 계획 중 기능을 명확히 구분합니다.
- 문서 내용이 코드와 충돌하면 코드를 확인하고 문서를 현재 구현과 합의된 방향에 맞게 정정합니다.
- 사용자가 `AGESTS.md`처럼 오타로 지칭해도 표준 파일명은 `AGENTS.md`를 유지합니다.

## 현재 기술 기준

- 언어: Kotlin
- UI: Android XML + Activity
- 지도: Kakao Maps SDK v2
- 네트워크: Retrofit + Gson + OkHttp
- JVM: Java 17
- Android SDK: `compileSdk 35`, `minSdk 24`, `targetSdk 35`
- 현재 화면: 메인, 현재 위치 기준 찾기, 목적지 기준 찾기, 설정

## 현재 위치 화면 흐름

1. 화면 진입 시 위치 권한과 GPS 위치를 확인합니다.
2. 위치 수신에 성공하면 현재 위치 마커와 5km 원을 표시합니다.
3. `GET /api/stations/nearby`로 반경 5km 주유소를 자동 조회합니다.
4. 응답을 지도 마커와 리스트에 반영합니다.
5. 첫 성공 응답 후 리스트는 반드시 하단 `최소화` 상태로 시작합니다.

- 조회 반경은 `radiusMeters=5000`을 사용합니다.
- 기본 유종은 `REGULAR_GASOLINE`, `PREMIUM_GASOLINE`, `DIESEL`입니다.
- 현재 위치와 백엔드 주유소 좌표는 WGS84의 위도·경도로 관리합니다.
- 주변 조회 기본 요청값은 현재 코드 기준 주유량 `30L`, 연비 `10km/L`, 정렬 `DISTANCE_ASC`입니다.
- 화면 리스트는 보통휘발유 가격이 낮은 순으로 다시 정렬합니다.

## 주유소 리스트 상태 계약

사용자에게 보이는 리스트 상태는 `최소화`, `절반`, `전체` 세 단계입니다.

| 문서 용어 | 코드 상태 | 정의 |
| --- | --- | --- |
| `최소화` | `MINIMIZED` | 화면 하단에 도크되어 손잡이와 헤더 일부가 보이는 상태 |
| `절반` | `COLLAPSED` | 화면 하단 절반을 차지하는 상태 |
| `전체` | `EXPANDED` | 화면 전체를 덮는 상태 |

- `HIDDEN`은 데이터가 아직 없거나 패널이 표시되지 않는 내부 상태입니다.
- `HIDDEN`을 최소화와 혼동하지 않습니다.
- 첫 주변 조회 성공 시 컨테이너를 `VISIBLE`로 만든 뒤 `stationSheetMinimizedTranslationY`에 배치하고 상태를 `MINIMIZED`로 설정합니다.
- 초기 최소화 표시는 기존 패널을 접는 `minimizeStationList()`의 숨김 방어 조건에 의존하지 않습니다.
- 최소화 상태에서 손잡이가 보이지 않으면 기능이 깨진 것으로 판단합니다.

### 리스트 상태 전환

| 현재 상태 | 위로 스와이프 | 아래로 스와이프 | 뒤로가기 |
| --- | --- | --- | --- |
| `최소화` | `절반` | 유지 | 첫 입력 안내, 2초 안에 두 번째 입력 시 메인 이동 |
| `절반` | `전체` | `최소화` | `최소화` |
| `전체` | 유지 | `절반` | `절반` |

- 손잡이 짧은 클릭은 `최소화 -> 절반`, `절반 -> 전체`, `전체 -> 절반`입니다.
- 아래로 스와이프는 현재 단계에서 뒤로가기와 같은 한 단계 축소 의미를 갖습니다.
- `전체` 상태에서 위로 스와이프해도 패널이 들썩이거나 상태가 바뀌면 안 됩니다.
- 최소화 상태는 일반 제스처나 한 번의 뒤로가기로 완전히 닫히지 않습니다.

### 리스트 터치 소유권

- 패널 상태 제스처는 `station_list_drag_handle_touch_area`와 손잡이에서만 받습니다.
- 손잡이 터치 영역은 화면 가로 전체, 높이 44dp를 기준으로 유지합니다.
- 리스트 본문은 항목 클릭과 RecyclerView 세로 스크롤만 담당합니다.
- 리스트 전체 컨테이너가 지도 터치를 가로채지 않게 합니다.
- 최소화와 절반 상태에서 배경 스크림은 `GONE`이며 클릭 불가 상태여야 합니다.
- 전체 상태에서 위쪽 지도 영역이 없는 것은 정상이며, 절반·최소화 상태의 지도 영역은 조작 가능해야 합니다.

### 리스트와 지도 버튼

- 리스트 또는 상세 패널이 표시되면 `현 위치로 설정` 버튼을 숨깁니다.
- GPS 버튼은 리스트 상태와 독립적으로 `VISIBLE`을 유지합니다.
- GPS와 줌 버튼의 elevation은 지도보다 높고 리스트·상세 패널보다 낮습니다.
- 따라서 지도 위에서는 보이지만 패널과 겹치는 부분에서는 패널 아래로 가려집니다.

## GPS 규칙

- 화면 진입 시 GPS를 자동으로 한 번 읽습니다.
- GPS 버튼을 누르면 저장된 현재 위치로 즉시 중심을 맞추고 줌을 15로 변경합니다.
- 이후 강제 GPS 재수신에 성공하면 새 현재 위치로 다시 중심을 맞춥니다.
- 이전 위치와 새 위치가 10m 미만 차이면 백엔드 재요청을 생략합니다.
- 10m 이상 차이면 새 좌표를 기준으로 주변 주유소를 다시 요청합니다.
- 현재 위치가 바뀌면 현재 위치 마커와 5km 원도 같은 좌표로 함께 이동해야 합니다.

## 지도 레이어 규칙

- 5km 원: 지도 바로 위, 모든 주유소 마커 아래
- 주유소 브랜드 마커: `station-icon-layer`, z-order 5000
- 주유소 유가 말풍선: `station-price-layer`, z-order 5001
- 현재 위치 마커: `current-location-layer`, z-order 6000
- GPS·줌 버튼: 지도 위, 패널 아래
- 리스트·상세 패널: 지도 버튼 위

- 현재 위치와 주유소 마커는 서로 다른 레이어에서 관리합니다.
- 주유소 마커와 말풍선은 `CompetitionType.None`으로 겹쳐도 숨기지 않습니다.
- 카메라 이동 종료 시 화면 Y 좌표를 기준으로 아래쪽 주유소가 높은 rank를 갖도록 정렬합니다.
- 기존 라벨을 가능한 한 재사용하고 필요한 항목만 추가·삭제해 깜박임을 줄입니다.

## 주유소 마커와 유가 규칙

- 지도 마커는 `drawable-nodpi/brand_*_short.png`를 사용합니다.
- 리스트와 상세 패널은 `drawable-nodpi/brand_*_full.png`를 사용합니다.
- 원본 로고는 `logo/`에 보관합니다.
- 로고는 투명 PNG를 사용하며 별도 배경을 추가하지 않습니다.
- 보이는 마커 크기는 유지하면서 투명 패딩으로 클릭 영역을 넓힐 수 있습니다.
- 유가 말풍선은 마커 위쪽에 두고 중앙 하단 꼬리가 마커를 가리키게 합니다.
- 말풍선 배경은 흰색으로 유지합니다.
- 가격이 0 이하인 유종은 표시하지 않습니다.
- 보통·고급휘발유가 모두 있으면 `휘발유(고급) 보통가격(고급가격)` 형식을 사용합니다.
- 하나만 있으면 해당 유종 이름과 가격만 표시합니다.
- 디젤은 별도 줄로 표시합니다.

### 줌별 렌더링

| 줌 | 표시 대상 |
| --- | --- |
| `11` 이하 | 최저가 top 5 |
| `12` | top 20 |
| `13~14` | top 50 |
| `15` 이상 | 전체 |

- top 순위는 보통휘발유 가격, 대표 가격, 거리, 이름 순으로 안정적으로 정렬합니다.
- 줌 15 이상에서는 디젤을 포함한 전체 가격 정보를 표시합니다.
- 더 낮은 줌에서는 휘발유 계열만 간소화해 표시합니다.
- 낮은 줌일수록 말풍선 투명도와 크기를 줄입니다.
- `+/-` 버튼은 `11 / 12 / 13 / 15`만 이동하며 핀치는 자유 줌을 유지합니다.

## 주유소 상세 패널 규칙

- 마커 또는 리스트 항목을 누르면 선택 주유소로 카메라를 이동하고 하단 상세 패널을 엽니다.
- 상세 패널은 처음 절반 상태로 열리고 위로 스와이프하면 전체 상태가 됩니다.
- 패널 밖을 누르거나 아래로 충분히 스와이프하면 닫힙니다.
- 상세 패널 스크림은 입력을 차단하지만 배경을 어둡게 하지 않습니다.
- 패널이 열린 동안 다른 지도 버튼보다 패널 입력을 우선합니다.
- 목록 데이터를 즉시 표시한 뒤 `GET /api/stations/{stationId}` 응답으로 갱신합니다.
- 표시 항목은 브랜드 로고, 상호명, 주소, 전화번호, 유가, 거리, 왕복 비용입니다.
- 전화번호가 비어 있으면 임시 안내를 표시합니다.
- 0원 유가는 표시하지 않습니다.
- 현재 왕복 비용은 응답 거리와 임시 연비 10km/L로 계산합니다.

## 경로 기능 상태

- `POST /api/stations/route` 요청 인터페이스와 repository 메서드는 존재합니다.
- 현재 `StationSearchResponse`에는 경로 형상 객체가 없으므로 Directions 경로 좌표를 받는 DTO 계약은 추가 구현이 필요합니다.
- `RouteInfo`와 `MapController.showRoute()` 인터페이스는 존재합니다.
- 현재 `KakaoMapController.showRoute()`는 데이터를 저장만 하며 실제 `RouteLine`을 렌더링하지 않습니다.
- 현재 주유소 선택 흐름은 상세 API만 호출하고 경로 API는 호출하지 않습니다.
- 따라서 문서에서 경로 조회·경로선·실제 경로 비용을 구현 완료로 표현하지 않습니다.
- 향후 구현 시 프론트는 현재 위치와 선택 주유소 좌표를 백엔드에 보내고, 백엔드는 네이버 Directions 5를 호출합니다.
- 프론트는 백엔드가 반환한 WGS84 좌표열을 Kakao `RouteLine`으로 그립니다.

## 목적지 화면 상태

- 목적지 화면은 풀스크린 Kakao 지도와 뒤로가기·GPS·목적지 설정 UI를 갖습니다.
- GPS 버튼으로 현재 위치를 읽고 지도 중심과 현재 위치 마커를 갱신합니다.
- 목적지 검색, 경로 요청, 경로 주변 주유소 조회는 아직 구현 전입니다.

## 백엔드 API 기준

- 주변 조회: `GET /api/stations/nearby`
- 상세 조회: `GET /api/stations/{stationId}`
- 경로 조회: `POST /api/stations/route`
- API 계약과 호출 예시는 `docs/FRONTEND_BACKEND_API_GUIDE.md`에서 관리합니다.
- 예전 `/api/stations/search/nearby` 경로는 사용하지 않습니다.
- Retrofit의 query list는 필요 시 Kotlin wildcard를 억제합니다.
- 현재 상세 API의 `phone` 값이 비어 있는 문제는 백엔드 이슈로 관리합니다.

## 백엔드 클레임 문서 규칙

- 백엔드 호출 실패마다 `docs/BACKEND_API_ACCESS_CLAIM.md`를 최신 내용으로 새로 작성합니다.
- 새 클레임을 쓰기 전에 기존 활성 문서를 `docs/archive/`로 옮깁니다.
- 활성 문서에는 요청 URL·메서드·파라미터 또는 JSON, 응답 코드·본문, 재현 환경, 프론트 분석, 백엔드 확인 요청을 포함합니다.
- 이전 실패 내용을 활성 문서에 누적하지 않습니다.
- 클레임과 아카이브 문서는 `.gitignore` 대상입니다.

## 비밀정보 규칙

- API 키, 네이티브 앱 키, 토큰, 서명키를 소스나 문서에 직접 작성하지 않습니다.
- Kakao 앱 키는 `secrets/kakao_native_app_key.txt`에서 읽습니다.
- 백엔드 기본 URL은 `secrets/backend_base_url.txt`에서 읽습니다.
- 실제 비밀 파일의 값은 읽거나 출력하지 않고 존재 여부만 확인하는 것을 기본으로 합니다.
- `secrets/*.txt`, `*.properties`, `*.key`, `*.jks`, `keystore.properties`, `signing/`은 Git에서 제외합니다.
- `secrets/*.example`과 `secrets/README.md`만 커밋 가능한 안내 자료입니다.
- release 빌드에는 Kakao 네이티브 앱 키를 포함하지 않습니다.
- 디버그 키 해시 로그는 개발 빌드에만 한정합니다.

## 개발 환경 연결

- Android Emulator 기본 백엔드 주소: `http://10.0.2.2:8080/`
- 같은 Wi-Fi 실기기: `http://<개발 PC LAN IP>:8080/`
- USB 실기기: `adb reverse tcp:8080 tcp:8080`과 `http://127.0.0.1:8080/`
- 외부 테스트: Cloudflare Tunnel HTTPS 주소
- 로컬 cleartext HTTP는 debug manifest에서만 허용합니다.

## 검증 규칙

- Kotlin 또는 Android 리소스를 수정하면 가능한 한 `.\gradlew.bat compileDebugKotlin`을 실행합니다.
- 설치와 실기기 동작이 중요한 UI 변경은 연결된 기기가 있으면 `.\gradlew.bat installDebug`까지 확인합니다.
- 주유소 리스트를 수정하면 최소한 다음을 확인합니다.
  - 첫 조회 성공 후 하단 최소화 패널과 손잡이가 보이는지
  - 최소화에서 위 스와이프 시 절반으로 열리는지
  - 절반에서 위·아래 스와이프가 각각 전체·최소화로 가는지
  - 전체에서 아래 스와이프와 뒤로가기가 절반으로 가는지
  - 리스트 본문 스크롤과 지도 터치가 서로 간섭하지 않는지
- 문서만 수정한 경우 빌드를 생략할 수 있으며 최종 응답에 생략 사실을 밝힙니다.

## Git 작업 규칙

- 사용자가 요청하지 않은 변경은 커밋하지 않습니다.
- 커밋 전에 `git status`와 diff를 확인하고 디버깅 이미지·UI dump·비밀 파일을 제외합니다.
- 커밋을 amend하지 않습니다. 사용자가 명시적으로 요청한 경우에만 예외입니다.
- push는 사용자가 요청한 경우에만 수행합니다.

## 커밋 메시지 규칙

- 문서 위주: `docs:`
- 기능 구현 위주: `feat:`
- 버그 수정 위주: `fix:`

형식:

```text
타입: 주된 핵심 변화

- 세부 변화 1
- 세부 변화 2
```

- 제목은 핵심 변화 한 줄로 작성합니다.
- 본문에는 세부 변경을 두 개 이상 bullet로 적습니다.
- 커밋 직전에 이 섹션을 다시 확인합니다.
