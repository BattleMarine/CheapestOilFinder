# CheapestOilFinder 1차 기획서

## 1. 제품 개요

### 제품명
CheapestOilFinder

### 한 줄 정의
주유소별 유가와 해당 주유소까지 이동하는 데 소요되는 연료비를 함께 계산하여, 실제 총비용 기준으로 가장 유리한 주유소를 추천하는 안드로이드 앱

### 문제 정의
사용자는 보통 "리터당 가격이 가장 싼 주유소"를 찾지만, 실제로는 그 주유소까지 이동하는 데 추가 연료가 소모된다.  
특히 먼 거리의 저가 주유소는 이동 비용까지 포함하면 오히려 손해일 수 있다.

기존 지도 앱이나 주유소 검색 서비스는 주로 아래 정보까지만 제공한다.

- 주유소 위치
- 리터당 가격
- 길찾기 정보

하지만 사용자가 실제로 궁금한 것은 아래에 가깝다.

- 지금 내 차 기준으로 어디에서 넣는 것이 가장 이득인가
- 목적지 가는 길에 들를 수 있는 주유소 중 어디가 가장 효율적인가
- 자주 다니는 동선에서 어떤 주유소를 우선적으로 봐야 하는가

본 앱은 이 차이를 해결하는 것을 목표로 한다.

## 2. 제품 목표

### 핵심 목표
- 사용자가 "가장 싼 주유소"가 아니라 "실제 총비용이 가장 낮은 주유소"를 빠르게 찾을 수 있게 한다.
- 현재 위치 또는 주요 목적지를 기준으로 현실적인 추천을 제공한다.
- 추후 사용자 주행 패턴을 반영하여 개인화된 추천으로 확장한다.

### 비핵심 목표
- 초기 버전에서 완전 자동 차량 진단 기능 제공
- 초기 버전에서 모든 차량/플랫폼과 깊게 연동되는 Android Auto 기능 제공
- 초기 버전에서 결제, 포인트 적립, 세차/정비 예약 등 부가 서비스 제공

- 유류비에 민감한 사용자

### 대표 사용자 시나리오

#### 시나리오 A. 주변 최적 주유소 탐색
사용자는 현재 위치 기준으로 주유소 목록을 확인하고, 자신의 차량 연비와 넣을 예정인 주유량을 입력한다. 앱은 각 주유소까지 이동하는 데 드는 연료비를 반영하여 실질적으로 가장 유리한 주유소를 추천한다.

#### 시나리오 B. 목적지 경유 기반 추천
사용자는 목적지를 입력한다. 앱은 현재 위치에서 목적지로 이동하는 경로 근처의 주유소를 우선 탐색하고, 우회 거리와 유가를 모두 고려하여 경로상 최적 주유소를 추천한다.

#### 시나리오 C. 반복 동선 기반 추천
사용자가 일정 기간 앱을 사용하면, 자주 가는 장소와 이동 시간대가 누적된다. 앱은 출퇴근길 또는 자주 가는 동선의 주유소에 가중치를 부여해 더 현실적인 추천을 제공한다.

## 6. 주요 기능 정의

### 6.1 주유소 탐색
- 현재 위치 기준 반경 내 주유소를 조회한다.
- 사용자는 반경 필터를 선택할 수 있다. 예: 3km, 5km, 10km <- 반경 필터를 적용할 때 거리 + 해당 거리를 이동하기 위해 필요한 연료 및 비용 (차량의 연비, 평균 유가 고려) 을 표기하도록 한다.
- 주유소별 브랜드, 가격, 거리, 예상 이동비용을 함께 표시한다.

### 6.2 비용 계산
사용자가 입력한 값을 기반으로 주유소별 총비용을 계산한다.

필수 입력값:
- 차량 공인 연비(km/L) <- 직접 입력 혹은 사전 차량 정보 등록 시 제조사 공인 연비를 이용
- 예상 주유량(L) <- 직접 입력 혹은 사전 차량 정보 등록 시 입력받도록
- 현재 위치 <- GPS 수신

선택 입력값:
- 목적지
- 현재 연료 잔량 또는 주행 가능 거리
- 경로 우선 여부

### 6.3 추천 엔진
추천 엔진은 각 주유소에 대해 아래 값을 계산한다.

- 주유 비용 = 주유소 가격 x 예상 주유량
- 이동 연료 소모량 = 주행거리 / 차량 연비
- 이동 연료비 = 이동 연료 소모량 x 기준 유가
- 총비용 = 주유 비용 + 이동 연료비 <- +알파로 유료도로 통과시 톨게이트 비용 또한 포함하도록

초기 버전에서는 기준 유가를 아래 중 하나로 단순화할 수 있다.

- 해당 주유소의 리터당 가격
- 전체 후보 주유소 평균 가격
- 사용자가 설정한 기준 연료비

추천 결과에는 최소 아래 항목을 노출한다.

- 최저가 주유소 (2순위)
- 총비용 기준 최적 주유소 (1순위)
- 가장 가까운 주유소 (3순위)

### 6.4 목적지 기반 추천
- 사용자가 목적지를 입력하면 현재 위치에서 목적지까지의 경로를 계산한다.
- 경로 주변 일정 거리 이내에 있는 주유소를 우선 후보군으로 삼는다.
- 경로 이탈 거리 또는 우회 시간을 반영해 우선순위를 계산한다.

### 6.5 사용자 설정
- 차량 연비 저장
- 연료 종류 저장
- 기본 예상 주유량 저장
- 지도 제공자 또는 계산 기준 선택 가능성 검토

## 7. 추천 알고리즘 초안

### 기본 계산식
후보 주유소 i에 대해

- 주유비(i) = 가격(i) x 예상주유량
- 이동거리(i) = 현재위치 -> 주유소 거리
- 이동연료소모(i) = 이동거리(i) / 차량연비
- 이동연료비(i) = 이동연료소모(i) x 기준단가
- 총비용(i) = 주유비(i) + 이동연료비(i)

### 목적지 포함 시
- 경로추가거리(i) = 현재위치 -> 주유소 -> 목적지 경로 - 현재위치 -> 목적지 직행 경로
- 경로추가연료비(i) = 경로추가거리(i) / 차량연비 x 기준단가
- 총비용(i) = 주유비(i) + 경로추가연료비(i)

### 향후 확장용 가중치
- 브랜드 선호도
- 자주 이용한 주유소 가산점 <- 가산점 보다는 따로 노출되도록
- 자주 가는 경로 인접 가산점
- 시간대별 선호 반영
- 우회 시간 패널티

### 알고리즘 설계 원칙
- 사용자가 계산 근거를 이해할 수 있어야 한다. <- 작은 버튼으로 산출 과정 수식 제공
- 추천 결과는 설명 가능해야 한다.
- 복잡한 AI 모델보다 명확한 규칙 기반 계산으로 시작한다.

- 위치 데이터는 민감정보에 준해 신중하게 다뤄야 한다.
- 수집 목적, 보관 기간, 사용자 동의 플로우가 필요하다.
- 개인화 기능은 옵트인 방식이 바람직하다.

## 13. 수익화 아이디어

## 15. 주요 리스크

### 15.1 유가 데이터 확보
- 가장 큰 리스크는 신뢰 가능한 유가 데이터 소스 확보 여부다.
- 지도 API만으로 충분한 가격 정보를 얻기 어려울 수 있다.

### 15.2 외부 API 비용
- 지도 및 경로 API 호출량이 증가하면 비용이 커질 수 있다. <- 사용자 별 조회 보다는 중앙 DB를 운영하여 프록시 서버의 형식을 취하는게 조회비용을 줄일 수 있을것
- MVP 단계에서 호출량 최적화 설계가 필요하다.

### 15.3 계산 신뢰성
- 사용자가 입력한 공인 연비와 실제 주행 연비 차이가 있을 수 있다.
- 추천 결과는 추정치임을 명확히 안내해야 한다.

### 15.4 Android Auto 확장성
- 규정, 권한, 플랫폼 지원 범위에 따라 구현 가능 범위가 달라질 수 있다.
- 1차 버전의 핵심 가치와 분리해서 검토해야 한다.

- 추천 계산 시 이동 연료비 기준단가는 무엇으로 둘 것인가

- 초기 타깃 지역을 전국으로 할지 특정 국가/시장으로 한정할지
- Android Auto를 실제 출시 목표에 포함할지, 연구 과제로 둘지

## 18. 제안 결론

이 서비스의 핵심은 "최저가 검색"이 아니라 "실질 총비용 최적화"에 있다.  
초기 버전은 안드로이드 모바일 앱으로 좁게 시작하고, 현재 위치 기반 추천과 목적지 기반 경로 추천을 핵심 가치로 삼는 것이 적절하다.

Android Auto 및 주행 데이터 기반 개인화는 충분히 매력적인 확장 방향이지만, MVP 단계에서는 외부 데이터 연동 안정성과 추천 계산 신뢰성을 먼저 확보하는 것이 우선이다.
