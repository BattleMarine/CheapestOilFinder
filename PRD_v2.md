# CheapestOilFinder 2차 기획서

## 1. 제품 개요

### 제품명
CheapestOilFinder

### 한 줄 정의
주유소별 유가, 이동에 드는 연료비, 통행료, 개인 할인 혜택까지 함께 계산해 실제 총비용 기준으로 가장 유리한 주유소를 추천하는 안드로이드 앱

### 제품이 해결하려는 문제
사용자는 보통 "리터당 가격이 가장 싼 주유소"를 찾지만, 실제 비용은 단순 가격 비교만으로 결정되지 않는다.  
주유소까지 이동하는 데 드는 연료비, 경로 이탈에 따른 시간 손실, 경우에 따라 발생하는 통행료, 그리고 카드/멤버십 할인까지 고려해야 실제로 어디가 가장 이득인지 판단할 수 있다.

기존 지도 앱이나 주유소 검색 서비스는 주로 아래 수준의 정보에 머무른다.

- 주유소 위치
- 리터당 가격
- 길찾기 정보

하지만 사용자가 실제로 원하는 것은 아래와 같다.

- 지금 내 차 기준으로 어디에서 넣는 것이 가장 이득인가
- 목적지 가는 길에 들를 수 있는 주유소 중 어디가 가장 효율적인가
- 내 할인 혜택까지 반영하면 어디가 최적인가
- 자주 다니는 동선에서 어떤 주유소를 우선적으로 보면 되는가

본 앱은 단순 가격 비교를 넘어서, 사용자의 실제 주유 의사결정에 가까운 계산 결과를 제공하는 것을 목표로 한다.

## 2. 제품 목표

### 핵심 목표
- 사용자가 "가장 싼 주유소"가 아니라 "실제 총비용이 가장 낮은 주유소"를 빠르게 찾을 수 있게 한다.
- 현재 위치 또는 목적지를 기준으로 현실적인 추천을 제공한다.
- 차량 정보, 할인 정보, 이동 비용을 반영한 설명 가능한 추천을 제공한다.
- 추후 사용자의 이동 패턴을 반영한 개인화 추천으로 확장한다.

### 비핵심 목표
- 초기 버전에서 차량 ECU/센서와 직접 연동되는 자동 진단 기능 제공
- 초기 버전에서 Android Auto 전용 앱 완성
- 초기 버전에서 결제, 포인트 적립 대행, 정비 예약 등 부가 서비스 제공

## 3. 대상 사용자

### 핵심 사용자
- 출퇴근 등 고정 이동 경로가 있는 운전자
- 유류비에 민감한 사용자
- 내 주변 최저가보다 실제 절약 금액이 궁금한 사용자
- 장거리 이동 전 합리적으로 주유하려는 사용자
- 통신사, 카드, 멤버십 할인 혜택까지 반영한 실제 체감 비용 비교를 원하는 사용자

### 대표 사용자 시나리오

#### 시나리오 A. 주변 최적 주유소 탐색
사용자는 현재 위치 기준으로 주유소 목록을 확인하고, 자신의 차량 연비와 예상 주유량을 기반으로 실질적으로 가장 유리한 주유소를 추천받는다.

#### 시나리오 B. 목적지 경유 기반 추천
사용자는 목적지를 입력한다. 앱은 현재 위치에서 목적지까지의 경로를 기준으로, 우회 거리와 경로 이탈 비용까지 고려해 최적 주유소를 추천한다.

#### 시나리오 C. 할인 반영 비교
사용자는 보유한 카드, 멤버십, 통신사 혜택을 등록한다. 앱은 표면 가격이 아니라 할인 적용 후 실질 가격으로 다시 계산하여 추천 결과를 조정한다.

#### 시나리오 D. 반복 동선 기반 추천
사용자가 일정 기간 앱을 사용하면 자주 가는 장소와 시간대가 누적된다. 앱은 반복 이동 패턴을 바탕으로 우선적으로 볼 만한 주유소를 따로 노출한다.

- 단순 가격 비교가 아니라 실제 절감 효과를 보여준다.

## 6. 주요 기능 정의

### 6.1 진입 화면
첫 화면은 과도한 정보 없이 두 가지 선택지만 제공한다.

- 현재 위치 기준으로 찾기
- 목적지 기준으로 찾기

사용자가 진입 방식을 고르면 다음 화면에서 즉시 추천 결과와 지도 또는 길찾기 연동 정보를 제공한다.

### 6.2 주유소 탐색
- 현재 위치 기준 반경 내 주유소를 조회한다.
- 반경 필터는 단순 거리값만 보여주는 것이 아니라, 해당 거리를 이동하는 데 필요한 연료량과 예상 이동 비용까지 함께 보여준다.
- 예: 3km, 5km, 10km 선택 시 "추가 이동 비용 예상"을 함께 표기
- 주유소별 브랜드, 가격, 거리, 예상 이동비용, 예상 총비용을 함께 표시한다.

### 6.3 비용 계산
사용자가 입력하거나 저장한 값을 기반으로 주유소별 총비용을 계산한다.

필수 입력값:
- 차량 연비(km/L)
- 예상 주유량(L)
- 현재 위치

입력 방식:
- 직접 입력
- 사전 차량 정보 등록 시 제조사 공인 연비 또는 사용자가 수정한 기본값 사용
- 기본 주유량 사전 저장
- 위치는 GPS 기반 수신

선택 입력값:
- 목적지
- 현재 연료 잔량 또는 주행 가능 거리
- 경로 우선 여부
- 할인 혜택 정보

### 6.4 추천 엔진
추천 엔진은 각 주유소에 대해 아래 요소를 계산한다.

- 주유 비용 = 주유소 가격 x 예상 주유량
- 이동 연료 소모량 = 주행거리 / 차량 연비
- 이동 연료비 = 이동 연료 소모량 x 기준 유가
- 통행료 = 해당 주유소 경유 시 추가로 발생하는 유료도로 비용
- 할인 반영 주유비 = 주유 비용 - 적용 가능한 할인/적립 효과
- 총비용 = 할인 반영 주유비 + 이동 연료비 + 통행료

초기 버전에서 기준 유가는 아래 중 하나로 단순화할 수 있다.

- 해당 주유소의 리터당 가격
- 전체 후보 주유소 평균 가격
- 사용자가 설정한 기준 연료비

추천 결과 기본 노출 순서는 아래와 같다.

- 총비용 기준 최적 주유소
- 최저가 주유소
- 가장 가까운 주유소

추가로 사용자가 자주 이용한 주유소는 점수에 직접 가산하지 않고, 별도의 배지 또는 섹션으로 노출한다.

### 6.5 계산 근거 설명
- 추천 결과는 설명 가능해야 한다.
- 각 주유소 카드 또는 상세 화면에서 작은 버튼을 눌러 산출 수식을 확인할 수 있어야 한다.
- 사용자는 "왜 이 주유소가 1순위인지"를 가격, 이동비, 통행료, 할인 항목별로 확인할 수 있어야 한다.

### 6.6 목적지 기반 추천
- 사용자가 목적지를 입력하면 현재 위치에서 목적지까지의 경로를 계산한다.
- 경로 주변 일정 거리 이내에 있는 주유소를 우선 후보군으로 삼는다.
- 경로 이탈 거리, 우회 시간, 통행료 발생 여부를 반영한다.
- 추천 결과에서는 경로상 추천 여부와 추가 우회 비용을 함께 보여준다.

### 6.7 사용자 설정
- 차량 정보 저장
- 연비 저장
- 연료 종류 저장
- 기본 예상 주유량 저장
- 할인 혜택 정보 저장
- 계산 방식 설정
- 지도 또는 길찾기 연동 방식 설정

## 7. 추천 알고리즘 초안

### 기본 계산식
후보 주유소 i에 대해

- 주유비(i) = 가격(i) x 예상주유량
- 이동거리(i) = 현재위치 -> 주유소 거리
- 이동연료소모(i) = 이동거리(i) / 차량연비
- 이동연료비(i) = 이동연료소모(i) x 기준단가
- 할인효과(i) = 카드/멤버십/프로모션으로 절감되는 금액
- 통행료(i) = 해당 경로에서 추가 발생하는 톨게이트 비용
- 총비용(i) = 주유비(i) - 할인효과(i) + 이동연료비(i) + 통행료(i)

### 목적지 포함 시
- 경로추가거리(i) = 현재위치 -> 주유소 -> 목적지 경로 - 현재위치 -> 목적지 직행 경로
- 경로추가연료비(i) = 경로추가거리(i) / 차량연비 x 기준단가
- 총비용(i) = 주유비(i) - 할인효과(i) + 경로추가연료비(i) + 통행료(i)

### 향후 확장용 보조 신호
- 자주 가는 경로 인접도
- 시간대별 사용 패턴
- 사용자가 실제 선택한 주유소 이력
- 만족도 피드백

주의:
자주 이용한 주유소는 추천 점수에 기계적으로 가산하지 않고, "자주 이용함" 또는 "익숙한 주유소" 같은 맥락 정보로 분리 노출하는 방향을 우선 검토한다.

### 알고리즘 설계 원칙
- 사용자가 계산 근거를 이해할 수 있어야 한다.
- 추천 결과는 설명 가능해야 한다.
- 복잡한 AI 모델보다 명확한 규칙 기반 계산으로 시작한다.
- 추정값과 확정값을 구분해서 안내한다.

## 11. Android Auto 및 확장 방향

### 장기 목표
- Android Auto 또는 Android for Cars 환경에서 주행 중 주유 관련 맥락 정보를 보조적으로 제공
- 주요 이동 경로 기반 개인화 추천 강화
- 주행 상황 인식 기반 UX 고도화

### 방향성 정리
초기에는 별도 Android Auto 전용 앱을 중심에 두기보다, 모바일 앱을 중심으로 시작하고 Android Auto 연결 상태 또는 차량 탑승 상황을 보조 신호로 활용하는 방향이 더 현실적이다.

예시:
- 차량 탑승 감지 시 빠른 추천 진입 제안
- 주행 시작 전 경로상 주유소 제안
- 주행 중에는 최소한의 정보만 제공

### 구현 현실성
- 차량 센서에 직접 접근하는 방식은 제약이 많다.
- 초기에는 GPS 기반 데이터와 사용자 입력 데이터 중심으로 설계하는 것이 현실적이다.
- Android Auto 관련 기능은 연구 및 확장 과제로 다루는 것이 적절하다.

### 현실적 접근 순서
1. 모바일 앱 MVP 출시
2. 목적지 기반 추천 고도화
3. 반복 이동 패턴 분석
4. 경로 기반 개인화 추천
5. Android Auto 보조 기능 검토

## 13. 수익화 방향

초기에는 제품 검증이 우선이며, 수익화는 사용자 신뢰를 해치지 않는 범위에서 검토한다.

우선순위가 낮은 방식:
- 추천 결과를 왜곡하는 제휴 우선 노출
- 사용성에 큰 영향을 주는 강한 광고 노출
- 핵심 기능 제한형 프리미엄 모델

검토 가능한 방향:
- 법인 차량/배달 기사 대상 B2B 기능
- 사용 흐름이 끝난 뒤 노출되는 소형 배너 광고
- 사용자 경험을 해치지 않는 수준의 제한적 광고

원칙:
광고나 제휴 모델이 추천 결과의 신뢰성을 훼손해서는 안 된다.

## 15. 주요 리스크

### 15.1 유가 데이터 확보
- 가장 큰 리스크는 신뢰 가능한 유가 데이터 소스 확보 여부다.
- 지도 API만으로 충분한 가격 정보를 얻기 어려울 수 있다.

### 15.2 외부 API 비용
- 지도 및 경로 API 호출량이 증가하면 비용이 커질 수 있다.
- 장기적으로는 중앙 DB와 프록시 서버 구조를 운영해 중복 조회를 줄이고 호출 비용을 낮추는 구조가 유리할 수 있다.
- MVP 단계에서부터 캐시, 조회 반경, 갱신 주기 설계를 고려해야 한다.

### 15.3 계산 신뢰성
- 사용자가 입력한 공인 연비와 실제 주행 연비 차이가 있을 수 있다.
- 할인 규칙이 복잡할 경우 계산 결과가 사용자 기대와 달라질 수 있다.
- 추천 결과는 추정치임을 명확히 안내해야 한다.

### 15.4 Android Auto 확장성
- 규정, 권한, 플랫폼 지원 범위에 따라 구현 가능 범위가 달라질 수 있다.
- 1차 버전의 핵심 가치와 분리해서 검토해야 한다.

### 15.5 UX 복잡도
- 할인, 연비, 경로, 통행료를 모두 한 번에 노출하면 초기 진입 장벽이 높아질 수 있다.
- 따라서 기본 사용 흐름은 단순하게 유지하고, 상세 계산은 확장형 UI로 제공해야 한다.

## 17. 오픈 이슈

기획 확정 전에 아래 항목의 의사결정이 필요하다.

- 주유소 가격 데이터는 어떤 출처를 사용할 것인가
- 추천 계산 시 이동 연료비 기준단가는 무엇으로 둘 것인가
- 통행료를 초기 버전부터 실제 계산에 포함할지, 구조만 마련할지
- 현재 위치 중심 앱으로 시작할지, 목적지 기반을 동시에 넣을지
- 사용자에게 현재 연료 잔량 입력을 받을지
- 할인 혜택은 어느 수준까지 지원할지
- 초기 타깃 지역을 전국으로 할지 특정 시장으로 한정할지
- Android Auto를 실제 출시 목표에 포함할지, 연구 과제로 둘지

## 18. 결론

이 서비스의 핵심은 "최저가 검색"이 아니라 "실질 총비용 최적화"에 있다.  
2차 기획 기준에서 이 제품은 단순한 주유소 검색 앱이 아니라, 사용자의 차량 조건과 이동 비용, 할인 혜택까지 반영한 의사결정 보조 도구로 정의된다.

초기 버전은 안드로이드 모바일 앱으로 시작하되, 첫 화면은 단순하게 유지하고 계산 엔진은 확장 가능하게 설계하는 것이 적절하다.  
특히 할인 혜택 반영, 계산 근거 설명, 외부 내비게이션 연동은 이 앱의 차별점을 만드는 중요한 요소다.

Android Auto 및 주행 데이터 기반 개인화는 충분히 매력적인 확장 방향이지만, MVP 단계에서는 신뢰 가능한 가격 데이터 확보, 계산의 설명 가능성, 사용 흐름의 단순성을 먼저 확보하는 것이 우선이다.
